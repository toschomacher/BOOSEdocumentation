    This command supports variable-based expressions for the radius. Expression evaluation is deferred until execution time in accordance with BOOSEâ€™s two-pass execution model.

    Extended behaviour: Supports an optional "filled" flag as the second parameter: circle radius, filled where filled can be true/false or 1/0.
